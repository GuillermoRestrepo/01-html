TIPOS DE <input>

Existe gran variedad de tipos de input, los cuales pueden ser utilizados al cambiar el valor del atributo (type=""), algunos de estos son:

<inputs> Para Botones

Existen 3 tipos principales de botones, los cuales pueden ser creados tanto con la etiqueta <input>, como con la etiqueta <button>, lo q marca la diferencia es la capacidad de personalización de cada etiqueta, siendo <button> la etiqueta q cuenta con más posibilidades al permitirse contener otras etiquetas. Los tipos de botones son:

  • type="submit": sirve para enviar enviar el contenido de los campos al servidor. (Es igual a (<butto type="submit">")
  • type="button": sirve para hacer botones que ejecuten eventos con JS. Es igual a (<butto type="button">")
  • type="reset": Limpia todos los campos del formulario y los deja como estaban originalmente. Sin enviar contenido al servidor. Es igual a (<butto type="reset">")

<inputs> Para Fecha Y Hora

Estos campos permiten a los usuarios seleccionar fechas, horas o ambas cosas de forma práctica, y son muy útiles en formularios que requieren agendar, reservar, programar, etc.

  • type="date": permite ingresar una fecha (Día / Mes / Año).
  • type="time": Permite ingresar una hora.
  • type="datetime-local": Permite ingresar una fecha y hora.
  • type="month": Permite ingresar mes y año.
  • type="week": permite ingresar el numero de una semana y un año.

Notas:

  • No es aconsejable utilizar type="datetime-local", ya que no todos los navegadores lo soportan, es mejor utilizar un campo de tipo fecha y uno de tipo hora por separado.
  • Los estilos que se presentan por defectos para los campos pueden ser modificados mediante el uso de CSS.

Atributos

  • max="": establecer una fechau Hora maxima
  • min="": establecer una fecha u hora minima
  • step="": establecer un salto de valores, ya sea en segundos minutos y dias.
  • value="": establecer un valor por defecto.
  • required="": establece un campo como obligatorio
  • readonly / disable: desactiva o bloquea el input.

<inputs> Para Moviles

Estos inputs tienen la cualidad de modificar el teclado de las pantallas de los moviles y las sugerencias de autocompletado.

  • type="search": Se usa para campos de búsqueda.
  • tytpe="email": Para ingresar direcciones de correo electrónico.
  • input type="tel": Para ingresar números telefónicos.
  • type="password":Para ingresar contraseñas. El contenido se oculta con puntos o asteriscos (•••••).
  • type="url": Para ingresar URLs o dire.Valida automáticamente que el texto comience con http:// o https://.

<inputs> Adicionales

son inputs q no guardan cierta relacion entre si ni con algun otro pero pueden ser útiles en diversas situaciones.

  • type="color": Muestra un selector de color (color picker) para que el usuario elija un color (formato hexadecimal: #RRGGBB).
  • type="number": Permite ingresar solo números. Puedes definir rangos, pasos, valores iniciales, etc. Atributos adicionales: (max, min, step, value)
  • type="range": Muestra una barra deslizante para seleccionar un valor numérico entre un rango. Atributos adicionales: (max, min, step, value)
  • type="reset": Reinicia todos los campos del formulario a sus valores iniciales (como estaban al cargarse la página).
  • type="text" Es el tipo más común. Se usa para cualquier tipo de texto libre: nombres, apellidos, direcciones, etc. Atributos adicionales: (maxlength, minlength, pattern, placeholder, required)
  • type="hidden": Sirve para guardar datos ocultos que no se muestran al usuario, pero que se envían al servidor cuando se envía el formulario. ejemplos: (Tokens de sesión, IDs de usuario, Valores generados automáticamente.)

<input type="file">

Es un campo de entrada que permite al usuario seleccionar uno o varios archivos de su dispositivo para subir al servidor.

Atributos

multiple
Permite seleccionar varios archivos a la vez.
accept=""
Filtra los tipos de archivo que se pueden seleccionar. Algunos valores son:(image/*, video/*, audio/*, .pdf, .doc, .docx, .jpg, .png)
required
Hace obligatorio seleccionar al menos un archivo.
capture
En móviles, puede activar la cámara o micrófono.

Es necesario q el formulaio tenga metodo POST para su correcto funcionamiento
