BOTONES EN FORMULARIOS HTML

En HTML existen dos formas principales de crear botones dentro de formularios: usando la etiqueta <input> con el atributo type y usando la etiqueta <button>. Aunque ambos pueden realizar acciones similares, tienen diferencias importantes.

1. <input> Como Botón

El elemento <input> puede comportarse como botón dependiendo del valor de su atributo type.

1.1. type="button"

  • Crea un botón sin acción predeterminada.
  • No envía el formulario.
  • Se usa normalmente con JavaScript.
<input type="button" value="Acción">

1.2. type="submit"

  • Envía el formulario.
  • Ejecuta la acción definida en action.
  • Es el botón de envío por defecto.
<input type="submit" value="Enviar">

1.3. type="reset"

  • Restablece todos los campos del formulario.
  • Vuelve a los valores iniciales.
  • Debe usarse con precaución.
<input type="reset" value="Restablecer">

2. Etiqueta <button>

La etiqueta <button> es más flexible que <input>, ya que puede contener texto, iconos y otras etiquetas.

2.1. type="button"

  • No envía el formulario.
  • Se usa para ejecutar acciones con JavaScript.
  • Es ideal para botones interactivos.
<button type="button">Acción</button>

2.2. type="submit"

  • Envía el formulario.
  • Es el comportamiento por defecto de <button>.
  • Debe declararse explícitamente para evitar errores.
<button type="submit">Enviar</button>

2.3. type="reset"

  • Restablece los valores del formulario.
  • Elimina los datos ingresados por el usuario.
  • Su uso no es común en formularios modernos.
<button type="reset">Restablecer</button>

3. Diferencias Entre <input> y <button>

Característica <input> <button>
Contenido interno Solo texto (value) Texto, iconos, HTML
Flexibilidad Limitada Alta
Uso recomendado Casos simples Formularios modernos

4. ¿Cuándo Usar Cada Uno?

  • Usa <button type="submit"> para enviar formularios.
  • Usa <button type="button"> para acciones con JavaScript.
  • Usa <input> solo cuando necesites simplicidad.
  • Evita reset si puede causar pérdida de datos.
