OTRAS ETIQUETAS UTILIZADAS EN FORMULARIOS

Además de los campos básicos como <input> y <select>, HTML incluye otras etiquetas diseñadas para mejorar la organización, accesibilidad y representación visual de datos dentro de formularios.

1. <fieldset>

Es un contenedor que agrupa campos relacionados dentro de un formulario.

  • Mejora la organización visual.
  • Aporta semántica al formulario.
  • Puede usarse junto con <legend>.
  • Puede deshabilitarse usando disabled.
<fieldset>
    <legend>Información Personal</legend>

    <label for="nombre">Nombre</label>
    <input type="text" id="nombre">

    <label for="edad">Edad</label>
    <input type="number" id="edad">
</fieldset>

2. <legend>

Proporciona un título o descripción al contenido del <fieldset>.

  • Se muestra como encabezado del grupo.
  • Mejora accesibilidad para lectores de pantalla.
  • Debe ser el primer elemento dentro del fieldset.
<fieldset>
    <legend>Datos de contacto</legend>
</fieldset>

3. <textarea>

Permite introducir texto de varias líneas.

  • Ideal para mensajes, comentarios o descripciones.
  • Puede ajustarse con rows y cols.
  • Permite redimensionarse dependiendo del navegador.
  • No es una etiqueta autocerrada.
<textarea id="mensaje" rows="4" cols="40">Escribe aquí...</textarea>

4. <meter>

Representa un valor dentro de un rango conocido, útil para indicadores.

  • Útil para niveles como batería, almacenamiento, reputación.
  • Requiere atributos como min, max y value.
  • No es una barra de progreso, sino un medidor.
<meter min="0" max="100" value="70">70%</meter>

5. <progress>

Muestra el progreso de una tarea en curso.

  • Ideal para cargas, descargas o procesos.
  • Puede mostrar progreso definido o indeterminado.
  • No representa un valor estático como meter.
<progress max="100" value="45">45%</progress>

Diferencias clave entre meter y progress

  • meter: indica un valor dentro de un rango conocido.
  • progress: indica el avance de una tarea que cambia con el tiempo.
