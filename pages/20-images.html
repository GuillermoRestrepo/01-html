USO DE IMÁGENES EN HTML

Las imágenes son elementos fundamentales dentro de cualquier página web. Permiten ilustrar contenido, complementar información y mejorar la experiencia visual del usuario. HTML ofrece varias etiquetas y atributos para trabajar correctamente con imágenes.

1. Etiqueta Principal: <img>

Es la etiqueta usada para insertar imágenes en una página web. Es una etiqueta autocerrada (no requiere cierre).

Uso Básico

<img src="imagen.jpg" alt="Descripción de la imagen">

Atributos Esenciales

  • src: Indica la ruta o URL de la imagen.
  • alt: Describe la imagen para accesibilidad y casos donde no carga.

Importancia Del Atributo alt

  • Permite a los lectores de pantalla interpretar la imagen.
  • Ayuda al posicionamiento SEO.
  • Muestra texto alternativo si la imagen falla o está bloqueada.

2. Atributos Adicionales Comunes

width y height

Establecen el tamaño de la imagen (en píxeles o porcentaje).

<img src="foto.jpg" width="300" height="200" alt="Foto">

title

Muestra un texto emergente cuando el usuario pasa el cursor por encima.

loading

Controla la carga diferida (lazy loading).

  • lazy: carga la imagen solo cuando está cerca de ser visible.
  • eager: se carga inmediatamente.
<img src="foto.jpg" loading="lazy" alt="Foto">

decoding

Indica cómo se debe decodificar la imagen.

  • async: recomendada, permite un rendimiento más fluido.
  • sync
  • auto

3. Etiqueta <picture>: imágenes adaptativas

Permite definir varias versiones de la misma imagen para diferentes tamaños de pantalla o resoluciones.

Estructura

<picture>
    <source srcset="imagen.webp" type="image/webp">
    <source srcset="imagen.jpg" type="image/jpeg">
    <img src="imagen.jpg" alt="Ejemplo">
</picture>

Ventajas

  • Permite usar formatos modernos (como WebP o AVIF).
  • Optimiza carga según el dispositivo.
  • Mejora rendimiento y SEO.

4. Formatos De Imagen Más Comunes

Cada formato tiene fortalezas y debilidades; elegir el adecuado depende de su uso.

PNG

  • Soporta transparencia.
  • Alta calidad, pero archivos más pesados.

JPG / JPEG

  • Formato comprimido con pérdida.
  • Ideal para fotografías.
  • Archivos ligeros.

GIF

  • Solo 256 colores.
  • Permite animación.
  • Desuso en muchos casos.

SVG

  • Gráficos vectoriales.
  • Escalables sin perder calidad.
  • Perfecto para iconos y logos.

WEBP

  • Excelente calidad.
  • Compresión muy alta.
  • Soporta transparencia.
  • Reemplazo moderno para PNG/JPEG.

AVIF

  • Mejor compresión y calidad que WebP.
  • Soporta HDR.
  • Formato más reciente.

5. ¿Cuál Es El Formato Más Recomendable?

Depende del caso:

  • WebP: mejor equilibrio entre calidad y peso → recomendado para casi todo.
  • AVIF: mejor calidad y compresión, pero menor soporte.
  • SVG: ideal para logos, iconos y gráficos simples.
  • JPG: útil para fotos si necesitas compatibilidad universal.
  • PNG: cuando necesitas transparencias o máxima nitidez.

6. Buenas Prácticas Al Usar Imágenes

  • Optimizar el peso antes de subirlas.
  • Usar alt descriptivo.
  • Preferir formatos modernos como WebP.
  • Evitar usar imágenes demasiado grandes.
  • Usar <picture> para responsive.
