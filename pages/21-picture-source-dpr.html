USO DE LAS ETIQUETAS <picture> Y <source>

HTML ofrece herramientas avanzadas para mostrar imágenes adaptadas a distintos dispositivos, resoluciones y condiciones. Las etiquetas <picture> y <source> permiten proporcionar varias versiones de una misma imagen, mejorando el rendimiento y la experiencia del usuario.

1. Etiqueta <picture>

La etiqueta <picture> funciona como un contenedor para definir múltiples recursos de imagen. Permite mostrar diferentes versiones según:

  • Ancho del dispositivo.
  • Formato soportado por el navegador.
  • Resolución de pantalla.

Estructura Básica

<picture>
    <source srcset="imagen.avif" type="image/avif">
    <source srcset="imagen.webp" type="image/webp">
    <img src="imagen.jpg" alt="Descripción">
</picture>

Ventajas Del Uso De <picture>

  • Permite imagen responsive avanzada.
  • Soporta cambio de formato según compatibilidad.
  • Optimiza rendimiento en dispositivos móviles.
  • Permite servir imágenes de alta resolución solo si el dispositivo lo necesita.

2. Etiqueta <source>

La etiqueta <source> se usa dentro de <picture> para definir condiciones específicas en las que se utilizará una imagen.

Atributos importantes

  • srcset: Lista de imágenes alternativas según resolución.
  • media: Permite usar imágenes basadas en media queries.
  • type: Indica el formato (image/webp, image/avif, etc.).

Ejemplo con media queries

<picture>
    <source media="(max-width: 600px)" srcset="imagen-pequena.jpg">
    <source media="(min-width: 601px)" srcset="imagen-grande.jpg">
    <img src="imagen-grande.jpg" alt="Imagen adaptativa">
</picture>

Ejemplo Con Múltiples Resoluciones

<picture>
    <source 
        srcset="foto-1x.webp 1x, foto-2x.webp 2x" 
        type="image/webp">

    <img 
        src="foto-1x.jpg" 
        srcset="foto-1x.jpg 1x, foto-2x.jpg 2x" 
        alt="Foto en alta resolución">
</picture>

3. Device Pixel Ratio (DPR)

El Device Pixel Ratio (DPR) es la relación entre los píxeles físicos de un dispositivo y los píxeles CSS. Indica cuántos píxeles reales se usan para dibujar un solo píxel lógico.

Ejemplos De DPR

  • DPR 1 → pantallas normales (monitores antiguos o básicos).
  • DPR 2 → pantallas Retina (muchos móviles y laptops).
  • DPR 3 o más → dispositivos de gama alta.

¿Por Qué Es Importante?

  • Permite mostrar imágenes más nítidas en pantallas de alta densidad.
  • Evita que las imágenes se vean borrosas.
  • Optimiza carga, sirviendo imágenes más pesadas solo cuando el dispositivo las necesita.

Uso De DPR Con srcset

<img 
    src="logo-1x.png" 
    srcset="logo-1x.png 1x, logo-2x.png 2x, logo-3x.png 3x" 
    alt="Logo responsive">

El navegador elegirá automáticamente la mejor imagen según la densidad de pantalla.

4. Combinando <picture> + srcset + DPR

Es posible mezclar formatos modernos, condiciones de tamaño y resoluciones avanzadas. Esto brinda el máximo nivel de optimización.

Ejemplo completo

<picture>
    <source 
        srcset="paisaje.avif 1x, paisaje@2x.avif 2x" 
        type="image/avif">

    <source 
        srcset="paisaje.webp 1x, paisaje@2x.webp 2x" 
        type="image/webp">

    <img 
        src="paisaje.jpg" 
        srcset="paisaje.jpg 1x, paisaje@2x.jpg 2x" 
        alt="Paisaje adaptativo">
</picture>

Beneficios

  • Imágenes más ligeras si el navegador soporta AVIF/WebP.
  • Compatible con navegadores antiguos.
  • Nitidez automática según densidad del dispositivo.
  • Mejor rendimiento y menor consumo de datos móviles.
