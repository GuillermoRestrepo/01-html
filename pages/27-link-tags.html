LA ETIQUETA <link> EN HTML

La etiqueta <link> es un elemento vacío (sin cierre) que se utiliza para definir relaciones entre el documento HTML y recursos externos. Su ubicación correcta es siempre dentro del elemento <head>.

¿Para Qué Se Etiliza La Etiqueta <link>?

Los usos más comunes son:

  • Vincular hojas de estilo CSS externas.
  • Definir iconos del sitio (favicon).
  • Cargar fuentes externas.
  • Definir recursos alternativos (como estilos alternativos).
  • Precargar, prerenderizar o preconectar recursos.

Atributos Principales De <link>

1. rel (relationship)

Define la relación del documento con el recurso. Es el atributo más importante.

Valores comunes:

  • stylesheet: indica que es una hoja de estilo CSS.
  • icon: define el favicon.
  • preload: permite precargar un recurso clave.
  • preconnect: prepara la conexión con un servidor externo.
  • alternate: define variantes del documento.
  • manifest: para aplicaciones web progresivas (PWA).

2. href

Especifica la URL del recurso indicado.

3. type

Indica el tipo MIME del recurso. No siempre es necesario.

4. sizes

Se usa en iconos para indicar su tamaño en píxeles.

5. media

Define condiciones para aplicar un recurso (como hojas de estilo para dispositivos específicos).

6. as

Usado cuando rel="preload" para indicar el tipo de recurso que se precarga.

Ejemplos De Uso

1. Vincular Un Archivo CSS

    <link rel="stylesheet" href="styles.css">

2. Usar Un Favicon

    <link rel="icon" href="favicon.ico" type="image/x-icon">

3. Precargar Recursos

    <link rel="preload" href="hero-image.jpg" as="image">
    <link rel="preload" href="main.css" as="style">

4. Preconectar Para Mejorar Rendimiento

    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

5. Cargar Fuentes Externas

    <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Roboto">

6. Hojas De Estilo Alternativas

    <link rel="alternate stylesheet" href="dark-mode.css" title="Modo Oscuro">

Buenas Prácticas

  • Colocar siempre la etiqueta dentro del <head>.
  • Optimizar el rendimiento usando preload y preconnect cuando sea necesario.
  • Evitar cargar hojas de estilo externas innecesarias.
  • Comprimir y minimizar los archivos vinculados (CSS, iconos, etc.).
  • Usar tamaños correctos para los iconos del sitio.

Errores Comunes

  • Intentar cerrar la etiqueta (</link>), lo cual es incorrecto.
  • Olvidar el atributo rel, haciendo que el navegador no sepa cómo usar el recurso.
  • Cargar demasiados estilos externos que afecten al rendimiento.
