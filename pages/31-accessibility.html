ACCESIBILIDAD EN HTML

La accesibilidad en HTML permite que las páginas web sean utilizables por todas las personas, incluyendo usuarios con discapacidades visuales, auditivas, cognitivas o motrices. HTML incorpora atributos especiales y buenas prácticas para mejorar la experiencia en lectores de pantalla, navegación por teclado y otras tecnologías asistivas.

1. Atributo alt En Imágenes

  • Proporciona una descripción textual de la imagen.
  • Es fundamental para lectores de pantalla.
  • Permite entender el contenido cuando la imagen no carga.
<img src="foto.jpg" alt="Persona usando un computador">

2. Atributo title

  • Muestra información adicional al pasar el mouse (tooltip).
  • No reemplaza a alt.
  • Puede servir para aclarar funciones de enlaces o botones.
<a href="descarga.pdf" title="Descargar archivo en PDF">Descargar</a>

3. Atributos ARIA (Accessible Rich Internet Applications)

Se utilizan para mejorar la semántica en elementos complejos o personalizados.

3.1. role

  • Define el propósito del elemento.
  • Útil cuando un elemento no semántico actúa como otro.
<div role="button">Abrir menú</div>

3.2. Atributos aria-*

  • aria-label: etiqueta accesible sin texto visible.
  • aria-labelledby: etiqueta accesible enlazada a otro elemento.
  • aria-describedby: agrega descripción adicional.
  • aria-hidden: oculta elementos a lectores de pantalla.
<button aria-label="Abrir menú de usuario">
    <img src="icono-menu.svg" alt="">
</button>

4. Navegación Mediante Teclado

4.1. Atributo tabindex

  • Define el orden de navegación por teclado.
  • 0: elemento entra en el orden natural.
  • -1: accesible por JavaScript, pero no por tabulación.
<div tabindex="0">Elemento enfocable</div>

5. Atributo lang

  • Indica el idioma del documento o fragmento.
  • Ayuda a los lectores de pantalla a pronunciar correctamente.
<p lang="en">Hello World</p>

6. Etiquetas Semánticas Para Accesibilidad

  • <header> — Encabezado
  • <main> — Contenido principal
  • <nav> — Navegación
  • <section> — Sección temática
  • <article> — Contenido independiente
  • <footer> — Pie de página

Estas etiquetas ayudan a los lectores de pantalla a entender la estructura del documento.

7. Contraste Y Texto Visible

  • Los enlaces deben ser distinguibles (color + subrayado).
  • El texto debe tener suficiente contraste con el fondo.
  • No depender solo del color para transmitir información.
